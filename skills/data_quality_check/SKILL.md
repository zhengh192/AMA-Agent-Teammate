# Data quality check

Run deterministic freshness, completeness, uniqueness, volume anomaly, schema consistency, referential integrity, and comparison-period coverage checks. Return `high`, `medium`, `low`, or `unusable` confidence with evidence. Stop dependent interpretation when a prerequisite fails or confidence is unusable. Never present high-confidence conclusions from low-confidence inputs.
